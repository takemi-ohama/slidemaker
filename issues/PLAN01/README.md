## プロジェクトサマリー

### 主要機能
- **新規作成モード**: MarkdownからAIでスライド生成
- **PDF/画像変換モード**: 既存スライドの再構築

### 技術スタック
- **言語**: Python 3.13、TypeScript
- **主要ライブラリ**: python-pptx、FastAPI、React
- **LLM**: Claude、GPT、Gemini等（マルチLLM対応）
- **インフラ**: AWS Lambda、API Gateway、S3

### 提供形態
- **CLI版**: PyPIパッケージ（`pip install slidemaker`）
- **WebUI版**: Webアプリケーション（AWS環境）

### 開発期間見積もり
- Phase 0-2（プロトタイプ）: 6-9週間
- Phase 3-4（CLI版完成）: 5-7週間
- Phase 5-6（WebUI版完成）: 5-7週間
- **合計**: 16-23週間（約4-6ヶ月）

## 開発優先順位

### 必須機能（MVP）
1. 新規作成モード（Markdown → PowerPoint）
2. LLM統合（Claude/GPT）
3. 基本的なスライド生成（テキスト・画像配置）
4. CLI版

### 優先度高
5. PDF/画像変換モード
6. 複数LLMサポート
7. WebUI版

### 優先度中
8. 高度なスタイリング
9. テンプレートシステム
10. 非同期処理

### 将来的な拡張
- Google Slides対応
- Keynote対応
- プレビュー機能
- 共同編集機能

## リスク管理

### 主要リスク
1. **LLM API制約**: 複数LLMサポートで緩和
2. **PowerPoint生成の複雑性**: 段階的実装で対応
3. **画像処理の精度**: プロンプト改善と手動調整機能
4. **Lambda制約**: メモリ最適化と非同期処理

詳細は **04_development_phases.md** のリスクセクションを参照。
